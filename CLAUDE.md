## Architecture

**Stack**: Next.js 16 App Router · TypeScript · Tailwind CSS v4 · Zustand · Supabase · PWA

### Data Flow

Each feature domain has a dedicated Zustand store in `store/` using `persist` middleware (localStorage). Supabase sync is **non-blocking**: mutations write to local state first, then fire-and-forget upserts to Supabase. On login, `fetchTodos`/`fetch*` merges Supabase rows with local state (Supabase wins on core fields; local wins on extended fields like `subtasks`, `tags`, `memo`).

### Auth

`lib/auth-context.tsx` provides `AuthProvider` and `useAuth()` hook. Auth state (user/session) lives in React context, not Zustand. Supabase OAuth callback handled at `app/api/auth/callback/route.ts`.

### Stores

| Store | Domain |
|-------|--------|
| `todoStore` | Todos with subtasks, kanban status, clips (work/personal) |
| `projectStore` | Projects with PRD, workflow status |
| `habitStore` | Daily habits/routines |
| `trackerStore` | Time/habit tracking |
| `noteStore` | Notes |
| `ideaStore` | Ideas |
| `linkStore` | Saved links |
| `newsStore` | News feed |
| `mindmapStore` | Mind maps |
| `agentStore` | Agent-related state |
| `uiStore` | UI state |

### Pages

All pages are under `app/` using the App Router. The shell layout (`app/layout.tsx`) wraps everything in `AuthProvider` and renders `<Sidebar>` + `<MobileHeader>`. Core feature pages: `/todo/daily`, `/todo/all`, `/board` (kanban), `/calendar`, `/routine`, `/tracker`, `/projects`, `/projects/[id]`, `/ideas`, `/links`, `/brainstorm`, `/mindmap`, `/news`, `/archive`, `/settings`.

### Database Schema

Defined in `lib/database.types.ts`. Only `todos` and `habits` tables currently sync to Supabase. Other store data lives in localStorage only.

### API Routes

- `app/api/auth/callback/route.ts` — Supabase OAuth callback
- `app/api/openai/generate-prd/route.ts` — GPT-powered PRD generation for projects

### PWA

Service worker at `public/sw.js`, registered via `components/ServiceWorkerRegistrar.tsx`. Icons generated by `scripts/generate-icons.mjs`.
